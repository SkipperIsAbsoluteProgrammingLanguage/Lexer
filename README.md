# Lexer + Parser для Skipper
Реализация фронтенда для C-подобного языка со строгой статической типизацией. Написана на **C#** с соблюдением принципов SOLID.

### 1. Lexer (Лексический анализатор)
Преобразует исходный код в поток токенов.
*   **Поддержка:** Ключевые слова, идентификаторы, литералы (int, float, bool, string, char), операторы и комментарии (однострочные/блочные).
*   **Диагностика:** Отслеживание точной позиции (строка, колонка) для каждого токена.
*   **Особенности:** Обработка escape-последовательностей в строках, распознавание многосимвольных операторов (`==`, `->`, `&&`).

### 2. Parser (Синтаксический анализатор)
Преобразует поток токенов в Абстрактное Синтаксическое Дерево (AST).
*   **Алгоритм:** Рекурсивный спуск (Recursive Descent).
*   **Выражения:** Поддержка приоритетов операций (Precedence Climbing), сложные логические и арифметические выражения.
*   **Конструкции:** Функции, классы, ветвления (`if/else`), циклы (`for`, `while`), переменные и блоки кода.
*   **Обработка ошибок:** Механизм **Panic Mode** (синхронизация по `;` или ключевым словам) для восстановления после ошибки и продолжения парсинга файла.

### 3. AST (Abstract Syntax Tree)
Иерархическая структура классов, описывающая программу.
*   **Структура:** Базовые типы `Declaration`, `Statement`, `Expression`.
*   **Расширяемость:** Реализация паттерна **Visitor** (`IAstVisitor`) для легкого добавления новых этапов обработки (семантический анализ, кодогенерация) без изменения классов узлов.

### 4. Тестирование (XUnit)
Полное покрытие функционала модульными тестами.
*   **Категории:**
    *   `LexerTests`: Проверка токенизации всех типов лексем.
    *   `ExpressionTests`: Проверка приоритетов операций и структуры выражений.
    *   `StatementTests`: Тестирование потока управления (циклы, условия).
    *   `DeclarationTests`: Проверка структур данных (классы, функции).
    *   `ErrorTests`: Валидация механизма восстановления после ошибок (Panic Mode).
